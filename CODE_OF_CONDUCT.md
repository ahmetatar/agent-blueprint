# Code of Conduct

## Our Standard

We want `agent-blueprint` to be a respectful, technically rigorous project. Participants are expected to contribute in a way that improves the work and keeps discussion constructive.

Examples of behavior that contributes to a productive environment:

- giving and receiving technical feedback respectfully
- focusing discussion on code, design, and evidence
- being explicit about tradeoffs and assumptions
- helping others understand project conventions and constraints

Examples of unacceptable behavior:

- personal attacks, harassment, or discrimination
- dismissive or hostile review behavior
- trolling, bad-faith argument, or intentionally disruptive conduct
- publishing private information without permission

## Scope

This Code of Conduct applies in project spaces, including issues, pull requests, discussions, and other community channels tied to this repository.

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, issues, and other contributions that violate this Code of Conduct.

Repeated or severe violations may result in temporary or permanent exclusion from project spaces.

## Reporting

Report unacceptable behavior privately to the maintainers through the contact path documented in `SECURITY.md` or through repository contact channels when available.

## Interpretation

This document is intentionally concise. Maintainers are expected to apply it with judgment, fairness, and a bias toward protecting contributors from abusive behavior while preserving direct technical discussion.
